# Map 3D — интерактивная карта и планировка вахтового городка

Веб-приложение для работы с картой в 3D (MapLibre + deck.gl), ручного и автоматического размещения модульных зданий внутри выбранной площадки, проверки противопожарных разрывов и обмена данными в формате GeoJSON.

## Возможности

| Область | Описание |
|--------|----------|
| **Карта** | Базовые стили CARTO (тёмная / светлая / voyager), опциональный рельеф (terrain), наклон камеры (pitch), масштаб |
| **Площадка** | Полигон строительной площадки: рисование, задание размера, применение; визуализация синей границей |
| **Генератор** | Автоматическая расстановка модулей вахтового городка (жилые, столовые, медпункт, КПП и др.) с учётом геометрии площадки и внутренних дорог |
| **Ручные объекты** | Добавление типовых зданий кликом, рисование зданий и дорог (Terra Draw), импорт / экспорт GeoJSON |
| **Пожарная безопасность** | Таблица разрывов по классам C0–C3 (СП 4.13130) в легенде; блокировка размещения нового здания, если нарушено расстояние между контурами существующих зданий |
| **Редактирование** | Режим выделения и перетаскивания зданий; удаление выбранного здания или всего сгенерированного «городка» |
| **Визуализация** | Обрезка отображаемых зданий и дорог по границе площадки (не «вылезают» за полигон на карте) |
| **Сохранение** | Локальное сохранение состояния раскладки в браузере (ключ `campLayoutState`) |

## Архитектура

Проект состоит из двух частей, которые в production обычно запускаются как **один процесс Node.js**:

1. **`app/map-3d-app`** — SPA на React + TypeScript + Vite. Собирается в статические файлы `dist/`.
2. **`app/backend`** — сервер Express, который:
   - отдаёт REST API с префиксом `/api`;
   - раздаёт содержимое `map-3d-app/dist` и для клиентских маршрутов отдаёт `index.html` (SPA fallback).

Схема запроса в production (типично за reverse proxy nginx):

```text
Браузер → nginx :80 → Node (Express) :PORT → статика + API
```

## Локальная разработка

### 1. Зависимости фронтенда

```bash
cd app/map-3d-app
npm ci
npm run dev
```

Vite поднимет dev-сервер (порт по умолчанию — см. вывод в терминале, обычно `5173`). В чистом dev-режиме бэкенд может не понадобиться, если вы не обращаетесь к API.

### 2. Сборка фронта и запуск полного стека локально

```bash
cd app/map-3d-app
npm ci
npm run build

cd ../backend
npm ci
```

По умолчанию в `server.js` используется `PORT` из окружения или **3001**:

```bash
export PORT=3001
node server.js
```

Откройте в браузере `http://localhost:3001`.

### 3. Линтинг фронта

```bash
cd app/map-3d-app
npm run lint
```

## Сборка для production

Порядок на сервере или в CI:

```bash
cd app/map-3d-app && npm ci && npm run build
cd ../backend && npm ci
```

Убедитесь, что каталог `app/map-3d-app/dist` существует и актуален — именно его читает `server.js`.

## Развёртывание на сервере

Ниже — обобщённый сценарий (Linux, root или отдельный пользователь).

### 1. Скопировать проект

Пример с `rsync` (исключить `node_modules` и при необходимости `dist`, если собираете на сервере):

```bash
rsync -avz --delete \
  -e "ssh -i ~/.ssh/your_key" \
  --exclude node_modules \
  ./app/ user@your-server:/path/to/app/
```

### 2. Установить зависимости и собрать фронт

```bash
ssh user@your-server
cd /path/to/app/map-3d-app && npm ci && npm run build
cd /path/to/app/backend && npm ci
```

### 3. Запуск через PM2

Важно задать **один и тот же** порт в переменной окружения и в `proxy_pass` nginx.

```bash
cd /path/to/app/backend
PORT=3000 pm2 start server.js --name map3d-app
pm2 save
pm2 startup   # выполнить предложенную systemd-команду
```

### 4. nginx (пример)

```nginx
server {
    listen 80;
    server_name your.domain.example;

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

Проверка: `nginx -t` и `systemctl reload nginx`.

### 5. Согласование порта

- В коде: `const PORT = process.env.PORT || 3001;`  
- Если PM2 запускает с `PORT=3000`, nginx должен проксировать на **3000**, иначе получите **502 Bad Gateway**.

## HTTP API

Базовый префикс: `/api`.

| Метод | Путь | Описание |
|--------|------|----------|
| `GET` | `/api/health` | Проверка живости: JSON `{ status, timestamp }` |
| `GET` | `/api/buildings` | Сейчас возвращает пустой `FeatureCollection` (заготовка под данные с сервера) |
| `POST` | `/api/upload-geojson` | Заглушка (реализация загрузки файла на сервер не завершена) |

Статические файлы и любые пути, не начинающиеся с `/api`, при наличии `dist` обрабатываются как SPA.

## Генератор вахтового городка и нормативы

В коде заданы правила, ориентированные на **ГОСТ Р 58760**, **СП 4.13130** (противопожарные разрывы между зданиями разных классов пожарной опасности **C0–C3**), санитарные разрывы для отдельных типов объектов, ширина проездов.

- Каталог модулей и параметры — в `Map3D.tsx` (константы `CAMP_RULES`, `MODULE_CATALOG` и связанная логика).  
- Автоматическая расстановка использует геометрию площадки, внутренние дороги генератора и проверки расстояний между контурами.  
- **Ручное** добавление зданий (клик или полигон Terra Draw) дополнительно проверяется функцией `validateFireBreaksForPolygon`: новое здание не добавляется, если нарушен разрыв или здание не целиком внутри площадки (если площадка задана).

Точные значения разрывов и пояснения отображаются в **легенде слева** («Пожароопасная зона»).

## Интерфейс: режимы работы

- **Просмотр** — панорамирование и масштаб карты; клики по зданиям для выделения не обрабатываются на слое deck.gl.  
- **Редактирование** — выделение здания кликом, перетаскивание; клавиши **Delete / Backspace** или кнопка удаления выбранного здания.  
- **Генератор** — блок в боковой панели: численность, состав, генерация / автоподбор площадки, сброс сгенерированного городка.  
- **Импорт / экспорт GeoJSON** — обмен сценой с объектами типа `site`, `building`, `road` (в зависимости от содержимого файла).

Подробные подсказки по шагам выведены в UI приложения.

## Ограничения и замечания

- Перетаскивание здания в режиме редактирования **не пересчитывает** противопожарные разрывы при каждом кадре — при жёстких требованиях имеет смысл доработать проверку на `dragend`.  
- `GET /api/buildings` и загрузка GeoJSON на сервер — **заготовки** для расширения.  
- Для работы 3D нужен браузер с **WebGL**; при ошибке инициализации включается упрощённый диагностический экран.  
- В репозиторий не стоит коммитить `node_modules` и при желании — локальные `dist`; собирайте артефакты в CI или на сервере.
